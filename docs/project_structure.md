# Project Structure
*This document defines the folder and file structure used in the project.*

## Folder Structure
```nano
./
├── README.md
├── requirements.txt
├── .gitignore
│
├── api
│   ├── app.py
│   └── db.py
│
├── data
│   ├── customers.csv
│   └── orders.csv
│
├── db
│   └── app.db
│
├── docs
│   └── project_structure.md
│
├── output
│
├── scripts
│   ├── db_bootstrap.py
│   └── etl_export.py
│
└── tests
```

### ./
The root project directory. All project files and directories are contained within this location.

### ./api
The api directory contains the implementation of the REST API created during Task 2.

Responsibilities include:
*  Creating the API server
* Connecting to the SQLite database
* Implementing endpoints
* Returning structured JSON responses

### ./data
The data directory stores the source datasets provided for the project.

These CSV files are used to populate the database during the bootstrap process.

Files:
* customers.csv
* orders.csv

These datasets contain 50 records each, meeting the project requirements.

### ./db
The db directory stores the SQLite database file used by the application.

The database is created automatically by the database bootstrap script.

File:
* app.db

This file should not normally be edited manually.

### ./docs
The docs directory contains project documentation related to the repository structure and implementation decisions.

Example documentation includes:
* project structure explanations
* design decisions
* implementation notes

### ./output
The output directory stores files generated by the ETL process in Task 3.

The export script writes processed CSV files to this directory.

### ./scripts
The scripts directory contains standalone executable scripts used for project tasks such as database setup and ETL processing.

These scripts are run manually and perform specific one-off operations.

### ./tests
The tests directory contains resources used for testing.

This may include:
* test scripts
* validation outputs
* logs
* manual testing utilities

## File Responsibilities

### requirements.txt
Lists all Python dependencies required to run the project environment.

### .gitignore
Defines files and directories that should not be committed to the repository (such as compiled files, environments, or generated artifacts).

## Core Components
### Database Bootstrap
> scripts/db_bootstrap.py

Responsibilities:
* Connect to the SQLite database
* Enable foreign key constraints
* Drop existing tables (if present)
* Create database schema
* Load CSV data from ./data
* Commit data to the database

This script prepares the database for use by the API and ETL processes.

#### API Service
> api/app.py

> api/db.py

Responsibilities:
* Create and run the API server
* Connect to the SQLite database
* Implement API endpoints
* Query database data
* Return responses in JSON format

Example endpoint:
```curl
GET /customers/<id>
```
Returns a customer and their associated orders.

api/db.py handles database connection and query logic, separating database access from API routing logic.

#### ETL Export
> scripts/etl_export.py

Responsibilities:
* Query active customers
* Join customers with orders
* Calculate order totals (quantity × unit_price)
* Create a full customer name field
* Export results to CSV in ./output

## Execution Flow
The expected workflow for running the project is:
1. Run the database bootstrap script to create the database and load the datasets
2. Start the API service to expose database data via endpoints
3. Run teh ETL export script to generate processed CSV outputs.

Example flow:
```
CSV datasets
     ↓
db_bootstrap.py
     ↓
SQLite database (app.db)
     ↓
API service / ETL script
     ↓
Output CSV files
```
