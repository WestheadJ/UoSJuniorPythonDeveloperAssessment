# Project Structure
*This defines the folder and file structure of the project*

## Folder Structure
```nano
./
├── README.md
├── api
├── data
├── db
├── docs
├── output
├── scripts
└── tests
```

### ./
This is the root project directory, everything should lie in here

### ./api
The `api` directory holds all the scripts and environment for the API made during Task 2

### ./data
The `data` directory holds the files for the datasets

### ./db
The `db` directory holds all relevant files and data linked to the database created in Task 1

### ./docs
The `docs` directory holds all documentation relating to the project apart form the project repo which will hold the project outline

### ./output
The `output` directory holds all the output files from the ETL script in Task 3

### ./scripts
The `scripts` directory holds all the main scripts, these are standalone executables, for example the bootstrap for the database

### ./tests
The `tests` directory holds all files and ouputs for testing, this can be logs, actual tests scripts or test outputs.
